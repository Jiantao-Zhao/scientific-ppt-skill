# Deck Skeletons by Genre

Every genre uses the **figure-first** style (`SKILL.md` + `reference.md`): each content slide built around large pasted figures, a navy conclusion title, one red take-home line, a citation. These skeletons give the slide *order*; the per-slide construction is always the same figure-first recipe. Backup/appendix slides go **after** the "Thank you" slide; detailed narration goes in speaker notes.

The two **gold-standard** genres are the English academic talk and the thesis-defense — match those first.

---

## 1. English academic talk / external seminar  ★ gold standard
For an audience partly outside the exact subfield — frame generously, every result on its own figure.

1. **Cover** — institution logo (top) + title + author / affiliation + group logo. Title may be navy or dark red.
2. **Outline** — short list of the talk's parts.
3. **Concept / background slides** — introduce the field with one scheme per idea (e.g. *"Click chemistry"*, *"Clip chemistry"*), each with a red one-line framing (*"…but are rarely reversible."*).
4. **Result slides** — one big figure each (scheme / EPR / UV / TA spectra / kinetics), **declarative title** stating the finding, red take-home line, citation. Annotate the key intermediate with a red box (e.g. *PTZ1•*).
5. **Mechanism / characterization** — supporting figures, sometimes a tiny I/II/III list of the logical steps.
6. **Summary & outlook** — a few red take-home lines (this is the one slide that may carry a short list).
7. **Closing thanks** — large; re-state name + affiliation.
8. **Backup** — extra spectra / kinetics for Q&A, after thanks.
Page numbers throughout; citation on every literature figure; full spoken script in notes.

---

## 2. Thesis defense / formal milestone  ★ gold standard
1. **Cover** — title + 答辩类型 + author / advisor / institution / date.
2. **目录 (Outline)** — numbered 一、二、三… of all sections.
3. **研究背景 (Background)** — motivation + the gap, figure-driven.
4. **Results — Part I** — section divider → result slides (one figure + finding title + red line each) → a **总结 (summary)** slide closing the part.
5. **Section divider → Results — Part II** — same pattern.
6. **总结与展望 (Conclusions & Outlook)** — design-principle take-homes.
7. **学术成果 (Publications)** — numbered list of own papers.
8. **致谢 (Acknowledgements)** — prose thanks.
9. **Thank-you** — *"谢谢大家，恳请批评指正"*.
10. **Backup** — supplementary data after the thank-you.
Each result section bracketed by a divider and closed by a 总结. Native tables OK for binding constants / redox data; everything else is figures with red-box annotation of key species.

---

## 3. Literature / paper sharing (文献分享)
1. **Cover** — journal masthead (screenshot) + paper title + presenter + date + "文献分享".
2. **作者简介 (Author bio)** — the paper authors' photo + Education / career. (A signature habit — introduce *who wrote it* before the science.)
3. **One slide per key result** — the paper's hero figure, titled with the specific finding, plus a small comparison table if useful; full citation.
4. **结论与启示 (Conclusions & implications)** — take-aways *for our lab*.
5. **Thank-you / backup.**

---

## 4. Periodic progress report (组会 / 3M quarterly / discussion)
1. **Title slide** — title + name + date (+ logo strip).
2. **(optional) recap** — one slide.
3. **Result slides** — figure-first, dated provenance tags (e.g. *"20240816"*). Data plots are usually pasted figures; use a native chart only per `data-mode.md`.
4. **Next steps / 下一步计划** — a few lines.
5. **Thank you** → backup (conditions, reproducibility, purity).

---

## 5. Self-introduction (自我介绍 / new group / interview)
1. **Cover** — "自我介绍" + name + date.
2. **个人简历 (CV / timeline)** — dated rows.
3. **Research-topic sections** — one short figure-first section per past topic.
4. **感兴趣的方向 (Future interests).**
5. **谢谢** → backup (detailed mechanism / characterization).

---

## 6. Poster / concept map
A single oversized canvas organized as panels around a central figure. Same palette/fonts; one clear visual narrative (left→right or center-out); native shapes/arrows for the schematic skeleton, pasted images for real scientific figures; headline finding as the banner.

---

## Choosing when unsure
Default to the **figure-first** style for everything. Only build a native data chart (`data-mode.md`) when the slide's point is a chart of Jiantao's own numbers and no good figure exists. Never produce a clean templated "dashboard" deck — that is not his style.
